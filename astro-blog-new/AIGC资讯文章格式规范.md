# AIGC资讯文章格式规范

## 1. Frontmatter (元数据)

```yaml
---
language: 'zh-CN'
title: '本周AIGC资讯 - 第X期'
description: '第一条新闻的摘要,作为文章描述'
publishDate: YYYY-MM-DD
tags:
  - AIGC资讯周刊
---
```

**说明**:
- `language`: 固定为 'zh-CN'
- `title`: 格式为"本周AIGC资讯 - 第X期"
- `description`: 使用第一条新闻的关键信息作为描述
- `publishDate`: 文章发布日期
- `tags`: 固定为 ["AIGC资讯周刊"]

## 2. 组件导入

```mdx
import { Card, Label } from 'astro-pure/user'
```

## 3. 文章结构

### 3.1 主标题
```mdx
# 本周AIGC资讯 - 第X期
```

### 3.2 新闻条目
每个新闻条目包含以下结构:

#### 3.2.1 二级标题
```mdx
## 新闻标题
```

#### 3.2.2 导语段落
```mdx
YYYY年M月D日,新闻导语内容...
```

#### 3.2.3 分节内容
使用三级标题划分不同部分:
```mdx
### 分节标题
具体内容...
```

#### 3.2.4 列表
使用无序列表列举要点:
```mdx
- 要点1
- 要点2
- 要点3
```

#### 3.2.5 来源标注
每个新闻条目末尾标注来源:
```mdx
**来源**: [来源名称](链接)、[来源名称](链接)
```

#### 3.2.6 分隔线
每个新闻条目之间使用分隔线:
```mdx
---
```

## 4. 总结部分

### 4.1 标题
```mdx
## 总结
```

### 4.2 关键动态列表
使用编号列表列举本周关键动态:
```mdx
本周 AIGC 领域的关键动态:

1. **第一条新闻标题**,简要说明
2. **第二条新闻标题**,简要说明
3. ...
```

### 4.3 总结性段落
一段文字总结本周整体趋势和意义。

### 4.4 分隔线和来源标注
```mdx
---

*资讯来源: 来源1、来源2、来源3*
```

## 5. 内容要求

### 5.1 时间范围
- 每期覆盖一周的资讯(周一至周日)
- 明确标注每条新闻的具体日期

### 5.2 新闻数量
- 建议每期包含8-12条重要新闻
- 涵盖不同领域:技术突破、行业动态、政策发布、市场数据、融资信息等

### 5.3 新闻类型
建议包含以下类型的新闻:
- **技术突破**: AI模型、算法、应用的新进展
- **行业动态**: 重大收购、合作、产品发布
- **政策发布**: 国家和地方相关政策
- **市场数据**: 用户规模、渗透率、市场份额等
- **融资信息**: 企业融资、投资动态
- **人才动态**: 招聘、培训、人才政策
- **会议活动**: 行业大会、论坛等

### 5.4 内容特点
- 信息准确,来源可靠
- 结构清晰,层次分明
- 语气客观,避免主观评价
- 每条新闻至少包含2个来源链接
- 突出关键数据和重要信息

## 6. 格式示例

完整的新闻条目示例:

```mdx
## 新闻标题

YYYY年M月D日,新闻导语内容...

### 分节标题1

具体内容...

### 分节标题2

- 要点1
- 要点2
- 要点3

**来源**: [来源名称](链接)、[来源名称](链接)

---
```

## 7. 注意事项

1. **日期格式**: 统一使用"YYYY年M月D日"格式
2. **标题层级**: 严格按照H1-H3的层级使用
3. **分隔线**: 每个新闻条目之间必须使用分隔线
4. **来源标注**: 每条新闻末尾必须标注来源
5. **链接格式**: 使用 Markdown 链接格式 `[名称](链接)`
6. **加粗强调**: 关键信息使用 `**文本**` 格式
