# 持续学习的习惯

> 技术这行，不学习就等于慢性自杀。但怎么才能坚持下去？

## 前言

昨天有个朋友跟我说："我想学新技术，但总是坚持不下来，怎么办？"

我问他："你每天花多少时间学习？"

他说："我想学的时候就学，没想学就不学。"

我笑了，说："那难怪坚持不下来。"

<Card>
**说实话**，我也经常想偷懒。但我知道，技术这行，不学习就等于慢性自杀。

这不是危言耸听，是真的。
</Card>

## 为什么必须持续学习

技术这行变化太快了，你不学习，别人就会超过你。

<Tabs>
  <TabItem label="技术更新快">
    前几年还在用jQuery，现在都React、Vue满天飞了。前几天刚学会的技术，可能过几个月就过时了。你不学，就会被淘汰。
  </TabItem>
  <TabItem label="竞争激烈">
    现在的程序员越来越多，你不学习，别人就会超过你。面试的时候，别人会的你不会，你就输了。
  </TabItem>
  <TabItem label="职业发展">
  想升职加薪？想跳槽涨薪？想成为技术大牛？这些都离不开持续学习。不学习，你就永远在原地踏步。
  </TabItem>
</Tabs>

## 我是怎么坚持学习的

说实话，我也不是天生就爱学习。但我找到了一些方法，让自己能坚持下去。

### 1. 制定学习计划

<Collapse title="我踩过的坑">
刚开始那会儿，我想学就学，想不学就不学。结果呢？一年下来，啥也没学会。

后来我意识到，**没有计划的学习，等于没有学习**。
</Collapse>

现在我会这么做：

<Label>每周制定计划</Label>
- 周日晚上，想好下周要学什么
- 把大目标分解成小任务（比如，"学习React"分解成"学习React基础"、"学习React Hooks"、"做一个小项目"）
- 每天完成一个小任务

<Label>固定学习时间</Label>
- 我每天早上8点到9点学习（早起一小时，没人打扰）
- 或者每天晚上8点到9点学习（下班后，先学习再娱乐）
- **关键是固定时间，养成习惯**

<Label>记录学习进度</Label>
- 用Notion或者简单的文本文件记录
- 每天完成一个任务，就打个勾
- 看着自己的进度，会有成就感

### 2. 培养学习兴趣

兴趣是最好的老师，这话一点没错。

<Card>
**举个例子**：我刚开始学Go的时候，觉得特别枯燥。后来我想了个办法——用Go写一个爬虫，爬我喜欢的网站。

一下子就有兴趣了！因为我想看到结果，所以学得特别快。
</Card>

怎么培养兴趣？

<Label>学感兴趣的</Label>
- 别强迫自己学不感兴趣的东西
- 找自己真正想学的，比如想做游戏就学游戏开发，想做网站就学前端
- 兴趣是最好的动力

<Label>关注技术趋势</Label>
- 看看现在流行什么技术
- 看看大厂在用什么技术
- 了解技术趋势，会激发学习欲望

<Label>参与技术社区</Label>
- 加一些技术群，和同行交流
- 参加技术meetup，认识志同道合的人
- 看别人的分享，会激发学习热情

### 3. 利用碎片时间

很多人说："我太忙了，没时间学习。"

其实，碎片时间也可以用来学习。

<Label>通勤时间</Label>
- 听技术播客（比如"代码时间"、"技术播客"）
- 看技术文章（用手机看，别刷抖音）
- 思考技术问题（比如，"这个功能怎么实现？"）

<Label>午休时间</Label>
- 看技术视频（B站上有很多好的视频）
- 读技术博客（推荐"掘金"、"知乎"）
- 写点代码（做个小demo）

<Label>等待时间</Label>
- 等地铁、等公交的时候，刷技术问答
- 等人的时候，看技术资讯
- 排队的时候，思考技术问题

### 4. 实践出真知

光看不练，等于白学。

<Card>
**我踩过的坑**：刚开始那会儿，我看了很多视频，读了很多文章，觉得自己学会了。结果一动手写代码，发现自己啥也不会。

后来我明白了，**理论结合实践，才能真正掌握**。
</Card>

怎么实践？

<Label>做项目</Label>
- 用学到的技术做个小项目
- 不用做得多复杂，能用就行
- 比如，学了React，就做个Todo List

<Label>写代码</Label>
- 多写代码，在实践中学习
- 别只看教程，要自己动手
- 遇到问题，自己解决

<Label>解决问题</Label>
- 在工作中遇到问题，自己先查资料
- 别一遇到问题就问别人
- 自己解决的过程，就是学习的过程

## 我的学习资源

这些年，我积累了一些不错的学习资源，分享给大家。

### 在线课程

<Label>视频课程</Label>
- **B站**：免费，内容多，质量参差不齐，需要筛选
- **慕课网**：国内比较专业的，有免费有付费
- **极客时间**：质量不错，但需要付费

<Label>互动课程</Label>
- **Codecademy**：边学边练，适合新手
- **freeCodeCamp**：完全免费，内容很全

### 技术书籍

<Label>经典书籍</Label>
- 《代码整洁之道》- 学会写好代码
- 《设计模式》- 学会设计
- 《算法导论》- 深入理解算法

<Label>实战书籍</Label>
- 《深入浅出Node.js》- Node.js实战
- 《React进阶之路》- React实战
- 《高性能MySQL》- 数据库优化

### 技术博客

<Label>个人博客</Label>
- 很多技术大牛都有自己的博客
- 关注一些大佬，学习他们的经验
- 比如"阮一峰的网络日志"、"美团技术团队"

<Label>技术社区</Label>
- **掘金**：质量不错，推荐
- **知乎**：有一些好的回答
- **CSDN**：内容多，但质量参差不齐

### 开源项目

<Label>阅读源码</Label>
- 看看优秀的开源项目是怎么写的
- 学习别人的设计思路
- 比如，看看Vue、React的源码

<Label>参与贡献</Label>
- 给开源项目提PR
- 在贡献中学习和成长
- 哪怕只是改个文档，也是学习

## 我的学习方法

这些年，我试过很多学习方法，总结了一些有用的。

### 费曼学习法

通过教别人来学习。

<Card>
**怎么做**：
1. 理解一个概念
2. 用简单的语言解释给别人听
3. 如果解释不清楚，说明自己没理解
4. 重新学习，直到能解释清楚

**我经常这么做**：学完一个技术，就写篇博客，教给别人。写博客的过程，就是自己学习的过程。
</Card>

### 主题学习法

围绕一个主题深入学习。

<Label>举个例子</Label>
- 我想学"高并发"
- 先收集资料：书、文章、视频
- 然后系统学习：从基础到高级
- 最后实践：做个高并发项目

### 问题驱动学习

通过解决问题来学习。

<Label>举个例子</Label>
- 工作中遇到性能问题
- 分析问题：为什么这么慢？
- 查找资料：怎么优化？
- 解决问题：实施优化方案
- 总结经验：写篇博客

### 项目驱动学习

通过做项目来学习。

<Label>举个例子</Label>
- 我想做个个人博客
- 技术选型：用React + Node.js
- 实现功能：边做边学
- 总结复盘：学到了什么

## 克服学习的障碍

学习路上，总会遇到各种障碍。我遇到过很多，也找到了一些解决办法。

### 时间不足

<Label>早起学习</Label>
- 早起一小时，没人打扰
- 一年下来，多了365小时的学习时间

<Label>利用碎片时间</Label>
- 通勤、午休、等待时间都可以利用
- 一天1小时，一年就是365小时

<Label>减少无效时间</Label>
- 别刷抖音、看短视频了
- 少刷朋友圈、微博
- 把这些时间用来学习

### 精力不足

<Label>保持健康</Label>
- 充足的睡眠（别熬夜）
- 适当的运动（每天运动30分钟）
- 健康的身体，才有精力学习

<Label>调整节奏</Label>
- 别过度学习，会累
- 每天学1-2小时就够了
- 关键是坚持，不是突击

<Label>选择合适时间</Label>
- 早上精力好，早上学
- 晚上精力好，晚上学
- 找到自己精力最好的时间

### 缺乏动力

<Label>设定目标</Label>
- 明确学习的目标
- 比如，"3个月内学会React，做个项目"
- 有了目标，才有动力

<Label>寻找伙伴</Label>
- 和志同道合的人一起学习
- 组个学习群，互相督促
- 有人一起学，更容易坚持

<Label>奖励自己</Label>
- 完成学习目标，给自己奖励
- 比如，学会React，奖励自己买个新键盘
- 适当的奖励，能激发动力

### 学习焦虑

<Label>接受现实</Label>
- 承认自己学不完所有东西
- 没人能学完所有技术
- 别给自己太大压力

<Label>专注重点</Label>
- 学习最重要的、最常用的
- 别什么都学，什么都学不精
- 20%的技术，解决80%的问题

<Label>循序渐进</Label>
- 一步一步来，别急于求成
- 先学基础，再学高级
- 别一开始就啃硬骨头

## 最后想说的话

持续学习是程序员的必备能力，但也是最难坚持的。

<Card>
**总结一下**：
- 制定学习计划，有计划的学习更有效
- 培养学习兴趣，兴趣是最好的老师
- 利用碎片时间，碎片时间也可以用来学习
- 实践出真知，理论结合实践

**记住**：
- 学习不是一蹴而就的，而是需要持续的努力
- 别给自己太大压力，慢慢来
- 重要的是坚持，不是突击
- 找到适合自己的学习方法
</Card>

<Collapse title="最后说个事儿">
我现在每天早上8点到9点学习，已经坚持了2年多。刚开始很难，但现在已经养成了习惯，不学反而觉得少了点什么。

如果你也想养成学习习惯，建议从小目标开始。比如，每天学30分钟，坚持一个月。习惯了，再慢慢增加时间。

学习这事儿，贵在坚持。💪
</Collapse>
